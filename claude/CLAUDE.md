* Be terse
* Always favour double quotes for strings
* Take a very sparse approach. Large blocks of code should be emitted only if absolutely necessary. Favour tight, small, modular code. 
* Comment only when it is not obvious what is going on in the code.
* Prefer line lengths of 100 characters or less, but use judgement on readability, and always prioritise correctness. 
* Major areas of related code in a single file should be demarcated with ----- xxxx ----- where xxxx is a short description of the code in that section.
